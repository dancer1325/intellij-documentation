## Render Javadocs

IntelliJ IDEA allows you to render Javadoc comments in the editor. Rendered comments are easier to read, and they do not overload your code with extra tags.

Click ![the Toggle Rendered View icon](https://resources.jetbrains.com/help/img/idea/2025.3/app-client.expui.gutter.javadocRead.svg) in the gutter next to the necessary documentation comment (or press `Ctrl+Alt+Q`) to toggle the rendered view; click ![the Toggle Rendered View icon](https://resources.jetbrains.com/help/img/idea/2025.3/app.expui.gutter.javadocEdit.svg) to edit the comment.

![Javadocs in the editing mode](https://resources.jetbrains.com/help/img/idea/2025.3/doc-comments-simple.png)

![Javadocs in the rendered mode](https://resources.jetbrains.com/help/img/idea/2025.3/doc-comment-rendered.png)

Rendered Javadoc comments allow you to click links to go to the referenced web pages or view [quick documentation](viewing-reference-information.html#inline-quick-documentation) for the referenced topics.

To change the font size, right-click a Javadoc comment in the editor and select Adjust Font Size from the context menu. Note that the rendered comments use the same font size as the quick documentation popup.

### Render Javadocs by default

You can configure the IDE to always render Javadocs in the editor.

  * Right-click the (![the Toggle Rendered View icon](https://resources.jetbrains.com/help/img/idea/2025.3/app-client.expui.gutter.javadocRead.svg) icon in the gutter or ![the Toggle Rendered View icon](https://resources.jetbrains.com/help/img/idea/2025.3/app-client.expui.gutter.javadocEdit.svg)) and enable the Render All option.

Alternatively, in the Settings dialog `Ctrl+Alt+S`, select Editor | General | Appearance and enable the Render documentation comments option.




To edit rendered Javadocs, click the ![the Toggle Rendered View icon](https://resources.jetbrains.com/help/img/idea/2025.3/app.expui.general.edit.svg) icon in the gutter next to the comment.
